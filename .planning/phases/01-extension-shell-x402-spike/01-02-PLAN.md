          - Define OG_CHAIN as: `{ id: 10744, name: "OpenGradient", nativeCurrency: { name: "OG", symbol: "OG", decimals: 18 }, rpcUrls: { default: { http: ["https://rpc.opengradient.ai"] } } }`

          - Import wrapFetch from "@x402/fetch", ExactEvmScheme from "@x402/evm/exact/client"

       b. `testLLMCall(x402Fetch: typeof fetch)`:
          - POST to "https://llmogevm.opengradient.ai/v1/chat/completions"
          - Body: model "openai/gpt-4o", messages [{ role: "user", content: "Respond with exactly: x402 connection verified" }], max_tokens 20, temperature 0
          - Return { success: boolean, content?: string, model?: string, usage?: object, error?: string }
          - Wrap in try/catch, return error details on failure

    2. Update entrypoints/background.ts to add message handlers (keep existing PING handler):

       a. 'SAVE_PRIVATE_KEY' handler:
          - Receives message.key
          - Stores it via `chrome.storage.session.set({ ogPrivateKey: message.key })`
          - Responds { success: true } or { success: false, error: ... }

       b. 'TEST_X402' handler:
          - Reads private key from `chrome.storage.session.get('ogPrivateKey')`
          - If no key, responds { success: false, error: 'No private key configured' }
          - Creates x402 client via createX402Client(ogPrivateKey)
          - Calls testLLMCall(x402Fetch)
          - Responds with the result
          - MUST return true from the onMessage listener to keep the channel open for async sendResponse

       c. Import createX402Client and testLLMCall from '@/lib/opengradient' (WXT path alias)

    3. Update entrypoints/popup/App.tsx to replace the placeholder UI:

       a. Add state: privateKey (string), keyStored (boolean), testing (boolean), result (TestResult | null)
       b. On mount (useEffect), check chrome.storage.session.get('ogPrivateKey') to see if key already stored
       c. If no key stored: show password input for private key (placeholder "0x...") and "Save Key (session only)" button
          - Validate key starts with "0x" and is 66 chars long before saving
          - After save, clear the input field immediately (don't keep the key in React state)
       d. If key stored: show "Wallet key configured" message and "Test x402 Gateway" button
       e. Show result panel below: green background for success (show content and model), red for failure (show error)
       f. Width: 380px, padding: 16px, font-family: system-ui
