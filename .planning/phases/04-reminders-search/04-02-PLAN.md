**lib/search.ts (NEW):**
Create the search utility module:
```
import Fuse from 'fuse.js';
import type { Task } from './types';
```

Define `fuseOptions: Fuse.IFuseOptions<Task>` with weighted keys:
- `{ name: 'action', weight: 2 }` -- primary: task action text (what to do)
- `{ name: 'context', weight: 1.5 }` -- secondary: context/details (person names, topics mentioned in extracted text)
- `{ name: 'type', weight: 0.5 }` -- tertiary: task type (call, meeting, etc.)
- `{ name: 'priority', weight: 0.3 }` -- low: priority level

The `context` key with weight 1.5 is critical for SRCH-04: it enables searching by person names and topics that appear in the extracted task context. For example, searching "John" will match a task whose context mentions "discussed with John about budget."

Additional options: `threshold: 0.4` (balanced fuzzy tolerance), `includeScore: true`, `minMatchCharLength: 2`.

Export `function searchTasks(tasks: Task[], query: string): Task[]`:
- If `!query.trim()`, return `tasks` unchanged (no filtering)
- Create `new Fuse(tasks, fuseOptions)` and call `.search(query)`
- Return `result.map(r => r.item)`

Note: Do NOT memoize the Fuse instance here -- that will be done in the React layer with `useMemo`.
