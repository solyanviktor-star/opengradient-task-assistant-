**entrypoints/background.ts:**
All new event listeners MUST be registered synchronously at the top level of the `defineBackground()` callback -- not inside async functions, not after awaits. This is critical for MV3 service worker lifecycle.

1. Add `const ALARM_PREFIX = 'reminder:';` at the top of the callback.

2. Register `chrome.alarms.onAlarm` listener at top level:
   - Guard: `if (!alarm.name.startsWith(ALARM_PREFIX)) return;`
   - Extract taskId from alarm name: `alarm.name.slice(ALARM_PREFIX.length)`
   - Read tasks from storage, find the task by ID
   - If task not found, return silently
   - Call `chrome.notifications.create(alarm.name, { type: 'basic', iconUrl: chrome.runtime.getURL('/128.png'), title: 'Task Reminder', message: task.action, priority: 2 })`
   - After showing notification, clear the `reminderAt` field: `await updateTask(taskId, { reminderAt: null })`
   - Note: Icon path is `/128.png` because WXT serves files from `public/` at the extension root.

3. Register `chrome.notifications.onClicked` listener at top level:
   - Guard: `if (!notificationId.startsWith(ALARM_PREFIX)) return;`
   - Extract taskId: `notificationId.slice(ALARM_PREFIX.length)`
   - Store `highlightTaskId` in chrome.storage.local: `await chrome.storage.local.set({ highlightTaskId: taskId })`
   - Try `chrome.action.openPopup()` with fallback to `chrome.tabs.create({ url: chrome.runtime.getURL('/popup.html') })`
   - Clear the notification: `chrome.notifications.clear(notificationId)`

4. Add `syncReminders()` async function:
   - Read all tasks from storage
   - Get all existing alarms via `chrome.alarms.getAll()`
   - Build a Set of existing alarm names
   - For each task with `reminderAt`:
     - If `when > Date.now()` and alarm doesn't exist: create it
     - If `when <= Date.now()`: clear the `reminderAt` field (expired)
   - Call `syncReminders()` from BOTH `browser.runtime.onInstalled` and a new `browser.runtime.onStartup` listener (register at top level).

5. Add `SET_REMINDER` message handler inside the existing `onMessage` listener:
   - Destructure `{ taskId, reminderAt }` from message
   - Call `await updateTask(taskId, { reminderAt })`
   - If `new Date(reminderAt).getTime() > Date.now()`: create alarm with `chrome.alarms.create(ALARM_PREFIX + taskId, { when })`
   - sendResponse `{ success: true }`

6. Add `CLEAR_REMINDER` message handler:
   - Call `await updateTask(message.taskId, { reminderAt: null })`
   - Clear alarm: `await chrome.alarms.clear(ALARM_PREFIX + message.taskId)`
   - sendResponse `{ success: true }`

Both message handlers follow the existing async pattern: wrap in `(async () => { ... })(); return true;`

7. Update the `EXTRACT_FROM_CLIPBOARD` handler: In the `enrichedTasks` construction (the `.map()` that builds `Task[]` from `rawTasks`), add `reminderAt: null` to the enriched task object alongside the existing `completed: false` and `completedAt: null` fields. This ensures newly extracted tasks have a valid `reminderAt` field from creation and satisfy the TypeScript `Task` interface.
