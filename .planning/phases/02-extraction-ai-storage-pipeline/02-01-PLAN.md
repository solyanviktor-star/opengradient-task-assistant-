**2. Create `lib/types.ts`** with the shared Task interface and message type definitions:

```typescript
export interface Task {
  id: string;                    // Deterministic: hash of sourceUrl + action text
  type: 'call' | 'meeting' | 'task' | 'note' | 'reminder';
  action: string;                // What to do
  deadline: string | null;       // ISO 8601 or null
  priority: 'low' | 'medium' | 'high';
  context: string;               // Additional details
  sourceUrl: string;             // Where extracted from
  platform: 'telegram' | 'selection' | 'other';
  createdAt: string;             // ISO 8601
  txHash: string | null;         // On-chain transaction hash from x402 payment
  memsyncId: string | null;      // MemSync memory ID after cloud sync
  synced: boolean;               // Whether synced to MemSync
}

// Message types for content script -> background communication
export interface ExtractTasksMessage {
  type: 'EXTRACT_TASKS';
  payload: {
    text: string;
    sourceUrl: string;
    platform: Task['platform'];
  };
}

export interface ExtractTasksResponse {
  success: boolean;
  tasks?: Task[];
  txHash?: string | null;
  error?: string;
}
```

Also export a `generateTaskId` function that creates a deterministic ID from sourceUrl + action text using a simple hash (use a basic string hash function -- no crypto dependency needed, this is just for dedup):

```typescript
export function generateTaskId(sourceUrl: string, action: string): string {
  const input = `${sourceUrl}::${action}`;
  let hash = 0;
  for (let i = 0; i < input.length; i++) {
    const char = input.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash |= 0; // Convert to 32bit integer
  }
  return `task_${Math.abs(hash).toString(36)}`;
}
```

**3. Create `entrypoints/content/telegram.content.ts`:**

Use WXT `defineContentScript` with `matches: ['*://web.telegram.org/*']` and `runAt: 'document_idle'`.

The content script should:
- Listen for `TRIGGER_EXTRACTION` messages from the background/popup
- When triggered, extract visible messages from the current Telegram Web A chat using a multi-strategy approach:
  - **Strategy 1 (preferred):** Query `[data-message-id]` elements, navigate to parent `.Message` ancestor, find `.text-content` within it, collect text content. This uses the stable data attribute as anchor.
  - **Strategy 2 (fallback):** Query `.text-content` elements directly inside `#MiddleColumn`.
  - **Strategy 3 (last resort):** Get innerText of `#MiddleColumn` or any `[class*="chat"]` container.
- After extracting text, send it to the background via `browser.runtime.sendMessage` with type `EXTRACT_TASKS` and payload `{ text, sourceUrl: window.location.href, platform: 'telegram' }`.
- Log extraction stats: how many messages found, which strategy was used.

IMPORTANT: Do NOT use MutationObserver or watch for live message changes -- extraction is user-triggered only (click extension icon). Keep it simple for MVP.

IMPORTANT: The content script sends the EXTRACT_TASKS message itself after extraction. The popup triggers extraction by sending TRIGGER_EXTRACTION to the content script (via `browser.tabs.sendMessage` from background, which will be wired in Plan 02-03). For now, the content script should also support being triggered directly by the user -- add a `browser.runtime.onMessage` listener for `TRIGGER_EXTRACTION`.

**1. Create `entrypoints/content/selection.content.ts`:**

This is the "second platform" that satisfies EXTR-05. Instead of a brittle Gmail scraper, implement a generic text selection extractor that works on ANY website.

Use WXT `defineContentScript` with `matches: ['<all_urls>']` and `runAt: 'document_idle'`.

The content script should:
- Listen for `TRIGGER_EXTRACTION` messages from the background/popup.
- When triggered, check if the user has selected text on the page via `window.getSelection()`.
- If text is selected (non-empty after trim), use that as the extraction text.
- If no text is selected, do nothing and respond with `{ text: '', url: window.location.href, platform: 'selection' }` (background will handle the empty case).
- Send extracted text to background via `browser.runtime.sendMessage` with type `EXTRACT_TASKS` and payload `{ text: selectedText, sourceUrl: window.location.href, platform: 'selection' }`.

This is intentionally simple. The power is in its universality -- it works on Gmail, Slack, WhatsApp Web, or any other page without platform-specific DOM selectors.

NOTE: The Telegram content script and selection content script will BOTH be injected on web.telegram.org. To avoid duplicate extraction, the TRIGGER_EXTRACTION message should include a `platform` field. The Telegram content script responds only to `platform: 'telegram'` triggers, and the selection content script responds only to `platform: 'selection'` triggers (or when no platform is specified and text is selected).
