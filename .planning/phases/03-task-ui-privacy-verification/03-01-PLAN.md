  <name>Task 1: Extend Task type and storage CRUD layer</name>
  <files>lib/types.ts, lib/storage.ts, entrypoints/background.ts</files>
  <action>
**lib/types.ts:**
1. Add `completed: boolean` and `completedAt: string | null` fields to the Task interface.
2. Remove `memsyncId: string | null` and `synced: boolean` fields -- MemSync was removed in Phase 2 (02-03). Old stored tasks with these fields are harmless (extra properties ignored at runtime).
3. Keep all other fields unchanged: id, type, action, deadline, priority, context, sourceUrl, platform, createdAt, txHash.

**lib/storage.ts:**
1. Add `updateTask(taskId: string, updates: Partial<Task>): Promise<void>` -- reads all tasks, maps over them replacing the matching task with spread `{...t, ...updates}`, writes back. If taskId not found, silently no-op.
2. Add `deleteTask(taskId: string): Promise<void>` -- reads all tasks, filters out the matching task, writes back.
3. In `getLocalTasks()`, add backward compatibility: after reading tasks, map each task to ensure `completed` defaults to `task.completed ?? false` and `completedAt` defaults to `task.completedAt ?? null`. This handles Phase 2 tasks that lack these fields.
4. Export all four functions: `saveTasksLocally`, `getLocalTasks`, `updateTask`, `deleteTask`.

**entrypoints/background.ts:**
1. Import `updateTask` and `deleteTask` from `@/lib/storage`.
2. Add `COMPLETE_TASK` handler: receives `{ type: "COMPLETE_TASK", taskId: string }`. Read the task via getLocalTasks to find it, toggle `completed` (flip boolean), set `completedAt` to `new Date().toISOString()` if completing or `null` if un-completing. Call `updateTask(taskId, { completed, completedAt })`. Respond `{ success: true }` or `{ success: false, error }`.
3. Add `DELETE_TASK` handler: receives `{ type: "DELETE_TASK", taskId: string }`. Call `deleteTask(taskId)`. Respond `{ success: true }` or `{ success: false, error }`.
4. In the `EXTRACT_FROM_CLIPBOARD` handler, update the enrichedTasks map to include `completed: false, completedAt: null` and remove `memsyncId: null, synced: false`.
5. Both new handlers must `return true` from the listener to signal async sendResponse.
